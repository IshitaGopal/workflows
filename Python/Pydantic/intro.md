# Pydantic 

- Used in data processing for ensuring clean data
- Used by Fast API 

## Type Hints vs Type Checking vs Data Validation:

1. Type Hints: A way to annotate variables, function parameters, and return values with expected types. This **does not** enforce types at runtime; it’s purely informational.

2. Type Checking: Detect type errors before runtime using tools like ``mypy``. This does **static analysis; no runtime effect** unless combined with runtime checks.

3. Data Validation: Actively checks that data meets certain criteria (types, ranges, formats, constraints) at runtime. **It enforce correctness at runtime**.


## Create a simple user model

- Gives the benefit of a nice syntax with type hints + data validation at runtime
- Collects all validation error at once 
- The ``username`` and ``email`` is required 
- ``bio`` and ``is_active`` are optional fields with defaults
- ``full_name`` can be a string or None and default is None (| = union)


```python 
# uv add pydantic
from pydantic import BaseModel, ValidationError
from datetime import datetime

class User(BaseModel):
    uid: int
    username: str
    email: str

    verified_at: datetime | None = None

    bio: str = ""
    is_active:bool = True
    full_name: str | None = None

```

## Accessing and modifying data

- You can access fields using dot notation.
- Model instances are mutable by default. 
- Models do not revalidate when you change a field. 

```python
user = User(uid=123,
            username = "igopal",
            email = "igopal@gmail.com")

print(user.username) # prints igopal

user.bio = 123 # Will run fine. Is not revalidating on assignment

```

## Different types 
- Supports all standard python types
- We can use unions, optionals, literal types
- Below we set a list of **tags** with defaults
    - ``default_factory``: function that is called to create a default each time you create an instance. Creates an empty list for each instance.
    -  We use a lamda function for datetime to return an unexecuted function that should run each time an instance is created 
    - partial() is the alternative to the lambda function.

```python
from pydantic import BaseModel, Field
from typing import Literal
from datetime import datetime

class BlogPost(BaseModel):
    titles: str
    content: str
    view_count: int = 0
    is_published: bool =False

    # Set list as default
    tags: list[str] = Field(default_factory=list)  

    # Pass an unexecuted function using lambda
    create_at: datetime = Field(default_factory = lambda: datetime.now(tz=UTC))
    # Another way to do the same thing
    create_at: datetime = Field(default_factory = partial(datetime.now, tz=UTC))

    # Union type 
    author_id: str | int 

    # Literal types 
    status: Literal["draft", "published", "archived"] = "draft"

```

## Using Fields and Annotated 

- Can be used to set defaults 
- Add constraints, like, min, max, patterns, linked requirements. The reccomended way to do this is to use Annotated type form ```Typing```
- Annotated adds metadata to our current type. For uid, we say: we want an integer *but* with the metadata that the value is greater than 0. 
- Pydantic has shorthand of constraints built in : (ref: 
1. https://docs.pydantic.dev/latest/api/types/#pydantic.types.NonPositiveInt
2. https://docs.pydantic.dev/latest/api/networks/)

- Some of these will have dependencies. Eg:, EmailStr requires the email validator library. (Do ``uv add pydantic[email]`` in terminal. []=optional dependencies)
```python
from typing import Annotated, Literal 
from datetime import datetime
from pydantic import BaseModel, Field, EmailStr, HttpUrl, SecretStr, ValidationError
from uuid import UUID, uuid4

class User(BaseModel):
    # int that is greater than 0
    uid: Annotated[int, Field(gt=0)]

    # Min and max length 
    username: Annotated[str, Field(min_length=3, max_length=20)]

    # age is where ge (or greater than or =) 13 and  le(less than or equal) <=130
    age: Annotated[int, Field(ge=13, le=130)]

    # Using regular expression pattern:
    #  should only contain lower case letters, numbers and hyphens
    slug: Annotated[str, Field(pattern="^[a-z0-9-]+$")]


```

```python
class User(BaseModel):

    # Make it uid with a default factory that generates a unique id -- autogenerated
    uid: UUID = Field(default_factory=uuid4)
```
